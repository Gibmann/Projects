# GitHub Projects Playground

This repository is a dedicated space for experimenting with **GitHub Projects** and refining my project management workflow. The goal is to bridge the gap between planning and execution using a structured, visual approach

## Workflow

To build solid habits, I mapped out a visual workflow in Figma. This repository puts that workflow into practice using a classic **Kanban** methodology

Here is how a task moves from an idea to completion:

### 1. Backlog
* **Action:** Create a GitHub Issue as soon as an idea or task pops up.
* **Goal:** Keep a clean, centralized record of everything to be done

### 2. Ready
* **Action:** Move the Issue here when planning the next work session
* **Goal:** Prioritize and commit to what is coming next

### 3. In Progress
* **Action:** Move the Issue here, create a dedicated branch, and start coding
* **Goal:** Keep focus on one active task at a time

### 4. In Review
* **Action:** Push the changes, open a **Pull Request**, and link the issue by adding `Closes #issue_number` in the description
* **Goal:** Double-check the work and prepare for integration

### 5. Done
* **Action:** Merge the PR. The linked Issue will close automatically. Delete the local/remote branch to keep the repo clean
* **Goal:** Celebrate progress and keep the workspace tidy
